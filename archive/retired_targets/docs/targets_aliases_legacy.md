# Histórico: aliases legacy de targets

> **Estado:** histórico, sin vigencia normativa.
> Este documento existe solo para trazabilidad y no amplía la política pública.

## Resumen

Antes de la política actual de targets se utilizaron alias y nombres no
canónicos que hoy deben considerarse retirados de la documentación pública,
ayudas de CLI, ejemplos y archivos de configuración.

## Inventario archivado

- `js` → usar `javascript`
- `c++` → usar `cpp`
- `assembly` → usar `asm`
- `ensamblador` → usar `asm`

## Regla vigente

La documentación normativa y los ejemplos públicos solo deben usar los targets
canónicos `python`, `rust`, `javascript`, `wasm`, `go`, `cpp`, `java` y `asm`.
